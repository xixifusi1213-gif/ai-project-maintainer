# v1.2.0: AI Agent Repair Pack

v1.2.0 turns gate reports into a repair workflow for AI coding assistants.

## Added

- New `repair-pack` command:

  ```powershell
  npx ai-project-maintainer repair-pack reports/security-report.json --project "E:\my-project" --output reports
  ```

- New output files:
  - `reports/fix-plan.md`
  - `reports/agent-tasks.json`
  - `reports/codex-tasks.json`
  - `reports/recheck-commands.ps1`
  - `reports/recheck-commands.sh`

- Generic `agent-tasks.json` task schema for AI coding assistants.
- Codex compatibility adapter in `codex-tasks.json`.
- Stable task types:
  - `auto_fix_candidate`
  - `needs_maintainer_decision`
  - `manual_review_required`
  - `recheck_only`

## Changed

- Public positioning no longer treats Codex as the only repair agent.
- README, skill instructions, report schema docs, and roadmap now describe the repair loop as a generic AI-agent workflow.
- Publish workflow now derives the GitHub Release title from the release notes heading instead of using a hardcoded title.

## Security

- Repair-pack output redacts token-like values, `Authorization`, `Bearer`, passwords, API keys, and DSNs.
- `needs_maintainer_decision` and `manual_review_required` tasks explicitly preserve maintainer risk ownership.

## Compatibility

- Existing `doctor`, `init`, `init-audit`, `audit-plan`, `gate`, `summary`, `connectors`, `evidence`, and `agent-risk` commands are unchanged.
- `passed` and `overallStatus` semantics are unchanged.
