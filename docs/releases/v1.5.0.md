# v1.5.0

v1.5.0 adds a production accident and data-exposure gate for AI-coded products.

## Added

- New production safety intake templates:
  - `.ai-maintainer/data-boundaries.yml`
  - `.ai-maintainer/authz-matrix.yml`
- Extended `.ai-maintainer/business-flows.yml` with side effects, abuse controls, idempotency, replay safety, and test evidence.
- New production audit checks for:
  - sensitive data boundaries and log redaction evidence;
  - authorization matrix and object-level authorization tests;
  - side-effectful business-flow idempotency and abuse controls;
  - database write safety and audit-log evidence;
  - AI repair safety boundaries.

## Behavior

- `quickstart` remains low-friction and report-only. It can show missing production safety model files as readiness gaps without turning them into first-run blockers.
- The full production gate can block missing data/auth/business-flow/database safety evidence when `production.block_on_coverage_gaps` is enabled.
- Repair-pack keeps production data and authorization gaps as maintainer-owned work, not AI-guessable fixes.

## Boundary

This is not a production safety guarantee or compliance certification. It turns common AI-coded production accident risks into explicit blockers, gaps, and maintainer decisions.
