# v1.4.2: Trivy DB resilience

v1.4.2 is a small first-run resilience patch for environments where Trivy cannot reliably download its vulnerability database.

## Changes

- Stop forcing a single GHCR Trivy DB repository when no mirror is configured.
- Allow `TRIVY_DB_REPOSITORY` and `TRIVY_JAVA_DB_REPOSITORY` to contain multiple comma or space separated repositories.
- Retry Trivy with `--skip-db-update` when the online DB update fails and a local cache may be available.
- Report cached-DB fallback as `PASS_WITH_GAPS` in non-strict quickstart, while strict release gates continue to block on incomplete DB freshness evidence.
- Show top evidence gaps in `quickstart-summary.md`, including Trivy cached-DB fallback notes.

## Boundary

Cached Trivy DB evidence keeps first-run usage from getting stuck. It is not the same as freshly updated release evidence. Refresh the Trivy DB from a reachable mirror before treating a strict release gate as complete.
