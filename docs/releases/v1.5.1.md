# v1.5.1

v1.5.1 makes gate results easier to interpret without adding scanners, connectors, or changing gate strictness.

## Added

- Additive `findingKind` classification for non-passing report checks and repair-pack tasks.
- A `findingSummary` count that separates confirmed vulnerabilities, untriaged scanner findings, verified check failures, production evidence gaps, maintainer decisions, and environment/tooling issues.
- Plain-language `What This Report Actually Found` sections in full and quickstart Markdown reports.
- Conservative SARIF metadata and filtering so production evidence gaps are not presented as code vulnerabilities by default.
- A real Ghost public-advisory before/after report image and a shorter README first-run path.
- A structured, redaction-first research workflow for 5-10 unfamiliar quickstart users.

## Behavior

- Existing blockers remain blockers; this patch changes explanation, not gate strictness.
- Scanner output is `untriaged_scanner_finding` unless validated evidence explicitly marks it as `confirmed_vulnerability`.
- Missing production proof remains `production_evidence_gap`.
- Tool or vulnerability-database availability failures remain `environment_tooling_issue`.

## Boundary

The report classification does not validate scanner findings automatically and does not turn a passing gate into a production safety guarantee.
