# v1.1.0: Project profile rule packs

This release adds profile-aware gates for different project types.

## Added

- `--profile auto` for `init`, `init-audit`, `audit-plan`, and `gate`.
- Profile rule packs:
  - `electron-desktop`
  - `nextjs-web`
  - `node-api`
  - `database-prisma`
  - `oss-library`
- Automatic profile detection with explicit CLI and policy overrides.
- Report fields:
  - `profile.id`
  - `profile.source`
  - `profile.signals`
  - `profile.riskFocus`
- Profile-specific audit-plan entries and wizard questions.
- Project profile documentation in English and Chinese.

## Compatibility

- Existing `profile: oss` and `--profile oss` values continue to work and map to `oss-library`.
- Existing report consumers remain compatible. The new `profile` field is additive.

## Validation

npm test
