# v1.3.1: Trust-language and Windows verification patch

v1.3.1 clarifies the public benchmark boundary and fixes published-release verification on Windows.

## Changed

- Clarified that benchmark cases use public incidents as evidence models.
- Added `Evidence type` to benchmark tables, summaries, and case metadata.
- Clarified that benchmark output does not modify upstream projects, vendor upstream source trees, ship exploit code, or claim upstream fixes were made by this tool.
- Updated release verification examples and scripts to `v1.3.1`.

## Fixed

- `verify-release.mjs --published` now resolves `npm` to `npm.cmd` on Windows while keeping shell-free command execution.
- Spawn creation errors, such as `ENOENT`, are now included in release verification failure output.

## Compatibility

- No CLI breaking changes.
- No new connectors, scanners, or external account requirements.
- `quickstart` remains planned for a later first-user-experience release.
