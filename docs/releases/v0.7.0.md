# v0.7.0: Production evidence connectors

This release adds optional read-only production evidence connectors.

## Added

- `connectors doctor` checks connector configuration without calling platform APIs.
- `evidence` generates `reports/evidence-report.json`.
- `gate --production --connectors` merges platform evidence into the production gate.
- GitHub Environments connector for required reviewers and deployment records.
- Sentry connector for error monitoring and release tracking.
- Vercel connector for production deployment records.
- Grafana connector for alert rule evidence.
- Prometheus connector for alerting rules and current alert state.
- Bytebase connector for database migration governance evidence.
- Atlas local migration lint connector.
- Cloudflare Pages, Render, and Fly deployment/runtime evidence connectors.
- `npm run smoke:connectors:live` for optional maintainer live validation.

## Security model

- Connectors are opt-in.
- Tokens are read only from environment variables.
- Reports redact token-like fields and never include Authorization headers.
- HTTP connectors use `GET` only.
- Atlas runs local lint and does not call Atlas Cloud.
- Missing token, 401/403, unavailable API, or missing local tool is reported as `GAP` by default.

## Notes

This release does not make external accounts mandatory. The default local and CI gate remains account-free unless `--connectors` is explicitly enabled.
