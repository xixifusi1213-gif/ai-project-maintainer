# v1.0.0: Trusted release chain and stable interfaces

v1.0.0 makes `ai-project-maintainer` a stable release readiness gate for AI-coded projects.

## Highlights

- Publishes through GitHub Actions and npm Trusted Publishing instead of local `npm publish`.
- Adds release manifest generation with SHA256 hashes for the npm tarball, SBOM, and security report.
- Adds release verification scripts for prepublish and published alignment checks.
- Adds GitHub artifact attestation for release artifacts.
- Documents the `1.x` report schema compatibility policy.
- Adds `SECURITY.md` and release trust documentation.

## Stable Commands

- `doctor`
- `init`
- `init-audit`
- `audit-plan`
- `gate`
- `summary`
- `connectors doctor`
- `evidence`
- `agent-risk`

## Notes

This release does not claim production safety or compliance certification. It makes the release evidence for the tool itself explicit and repeatable.
