# v1.4.0: Quickstart first-run experience

v1.4.0 adds a low-friction first-run path for people trying AI Project Maintainer for the first time.

## Added

- New `quickstart` command:
  ```powershell
  npx ai-project-maintainer quickstart .
  ```
- Report-only first-run workflow that writes under `reports/` without creating `.ai-maintainer` or CI files.
- Lightweight gate defaults for quickstart:
  - auto profile detection
  - AI agent risk checks enabled
  - tests skipped
  - strict, release, production evidence, and connectors disabled
- Short quickstart summary:
  - `reports/quickstart-summary.md`
  - `reports/quickstart-summary.json`
- Quickstart security report:
  - `reports/quickstart-security-report.json`
  - `reports/quickstart-security-report.md`
  - `reports/quickstart-security-report.sarif`
- Automatic `reports/quickstart-repair-pack/` generation when blockers exist.

## Changed

- README, install docs, and demo docs now present `quickstart` as the first command for unfamiliar users.
- Release verification scripts and examples now target `v1.4.0`.

## Compatibility

- Existing `doctor`, `init`, `init-audit`, `audit-plan`, `gate`, `repair-pack`, and connector commands are unchanged.
- No new scanners, connectors, accounts, or credentials are required.
- `quickstart` returns success when it completes and writes reports, even if the generated report contains blockers.
