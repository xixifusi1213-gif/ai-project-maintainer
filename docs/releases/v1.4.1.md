# v1.4.1: First-run environment resilience

v1.4.1 is a small patch for the `v1.4.0` quickstart release. It improves first-run behavior without adding connectors, scanners, or benchmark cases.

## Changes

- Quickstart now treats npm `ENOLOCK` in a brand-new project as a setup/evidence gap when `package.json` exists but no npm lockfile exists.
- In that case, quickstart reports `PASS_WITH_GAPS` when there are no other blockers and does not generate `reports/quickstart-repair-pack/`.
- The quickstart summary now includes a setup note telling users to run `npm install --package-lock-only` before relying on npm audit evidence.
- Full release gates keep stricter behavior and can still require dependency audit evidence.
- Trivy database download failures continue to be reported as incomplete scanner evidence, not as discovered vulnerabilities.
- Local skill docs now present `npx ai-project-maintainer quickstart .` as the first command.
- Direct local script users get clearer guidance when the local skill installation is missing the `yaml` dependency.

## Boundaries

- No private local project names, paths, or reports are included.
- No new scanners, connectors, CI scaffolding, or benchmark categories were added.
- `v1.4.0` quickstart remains the main promotion message; this patch only reduces environment friction.
