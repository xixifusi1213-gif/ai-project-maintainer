# v1.4.4: Windows release verify shim

v1.4.4 is a small Windows compatibility patch for local release verification after v1.4.3.

## Changes

- Fix `release:verify:published` on Windows when direct `spawnSync("npm.cmd")` returns `EINVAL`.
- Route only Windows npm verification calls through `cmd.exe /d /s /c call npm.cmd ...`.
- Keep `spawnSync` shell-free and leave `git`, `gh`, and non-Windows command resolution unchanged.
- Keep the v1.4.3 quickstart Semgrep calibration unchanged.

## Boundary

This release does not change scanner policy, quickstart severity rules, connectors, or benchmark behavior. It only makes local Windows release verification match the already-successful GitHub Actions trusted publishing path.
